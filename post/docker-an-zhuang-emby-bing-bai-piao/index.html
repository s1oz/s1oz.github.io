Docker安装Emby并白嫖

发布于 2020-03-26 | 分类于 docker 、 emby | 6分钟 | 1460字数

聊聊

海报墙软件好多,但是哪个好用呢,都用过来,我觉得还是Emby比较好时😀
Emby:
相对成熟的方案,全平台支持,白嫖后没啥缺点了,感觉缺点可能就是不能直接播放ISO文件跟国产剧命名吧,不过对于我这种只下REMUX的来说不是问题,国产剧..硬链接批量加S1呗~
ISO文件真要看,大不了装kodi插件呗~
Jellyfin:
基于Emby3.x的开源代码二次开发,相对而言,不能这样说,APP就是垃圾!!
优点就在于免费,10.7.x开始支持ISO文件直接播放,不过app字幕有点问题,刮削反正都用的同样的源,我是没遇到其他网友说的EMBY刮不出.jellyfin好刮的问题,这种问题我觉得一律归为蹄子不香
现在的APP不香,有兴趣的可以用测试版,地址在 这 优点是界面改了,好看多了,但是使用体验....你懂的
Plex:
跟emby一样,硬解之类功能需要收费,不过便宜,淘宝大概300+的终身key,(感觉都是黑卡搞来的)
缺点在于,这憨憨远程访问要过自家服务器验证,对于访问plex服务器不畅通的选手来说,体验不佳
至于界面呢,跟EMBY完全不同,萝卜青菜各有所爱嘛~其他也没啥缺点
个人感觉Plex是专做流媒体了,emby反正像个本地海报墙
KODI:
我是用不惯,对于海报墙来说,上述几个都比这个好看,可能有点就在于本地直接装一个吧,适合就在电视上安装,刮削nas上的电影,反正也没播放记录同步
群辉VIDEOSTATION:
一开始就是被这玩意儿拉入坑的,额外加持刮削补丁后海报墙体验还不错,尤其对于国产剧集,算是不改名的最优解但是APP呢,不咋滴,个人感觉是不咋滴,支持格式少,要么就安装额外的如mxplayer之类调用播放,
Infuse:
好用是好用,你丫的有本事支持下安卓呀,所以.不说了
可以作为插件支持前面三种,直接读取刮削数据,香
垃圾infuse,安卓都不支持
所以哪个合适自己就选择哪个咯😉

准备嫖她

以Unraid为例

安装User Scripts插件
将dll ca.7z解压后移动到unraid中如我存放的路径/mnt/user/appdata/ca/
在User Scripts插件中新建名为emby的脚本,并设置为At First Array Start Only

点击EDIT SCRIPTS加入以下内容

!/bin/bash
cp -rf /mnt/user/appdata/ca/default.conf /etc/nginx/conf.d/default.conf
nginx -s reload

APPLY保存 再点击RUN SCRIPTS运行一次脚本

在路由中劫持验证服务器地址

OP为例 hosts中添加以下信息

  192.168.1.10 mb3admin.com
  192.168.1.10 www.mb3admin.com

如果有IPV6的,要么勾选禁止解析 IPv6 DNS 记录

要么就hosts中加入ipv6地址劫持

应用中搜素Emby,选择EmbyServerBeta (BETA)安装 🤞反正我是喜欢追新
里面根据自己片子目录映射,需要注意的就是两点

然后点击应用即可,以后如果官方更新,直接点更新即可,完全不影响,这样即可完美追新

其他平台咋办

都一个意思,docker通用,打补丁,劫持到伪站验证
群辉套件之类的..已经有一键脚本了.转docker吧,追新才是王道
群辉利用Web Station搭建验证伪站可以替换此文件***user.conf***
路径为

/usr/local/etc/nginx/conf.d/*-*-*-*

*-*-*-*为建立虚拟主机的随机数字的文件夹
替换后以此命令刷新

nginx -s reload

核显调用

现在的人,永远就自动照搬,不看官方更新,请不要在go文件中添加

    modprobe i915

因为从beta35开始,官方就给出了此方法

    mkdir /boot/config/modprobe.d
    touch /boot/config/modprobe.d/i915.conf

Unraid 默认会屏蔽显卡驱动,所以通过新建同名文件的方法来解除屏蔽,这样创建后,开机默认会加载核显
请不要在go文件中添加这个了

chmod -R 777 /dev/dri

6.9.1开始,默认会给容器中调用显卡设置为0777权限了
所以只需要选择添加设备,填入/dev/dri即可

Emby服务器没有小黄标

请自行排查mb3admin.com网址劫持到unriad所搭建的验证伪站
一般都是犹豫默认解析ipv6域名造成的,禁用或者添加解析即可

windows浏览器没有小黄标

windows需要安装刚ca.7z解压文件中的guomi.cer证书

IOS设备无法正常使用

IOS需要安装后在设置--通用--关于手机--证书信任设置中把guomi.cer证书信任
