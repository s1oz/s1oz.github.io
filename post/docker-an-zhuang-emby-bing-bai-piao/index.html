搭建伪站

方法1 以Unraid自带nginx为例 仅适用小于等于6.9.2版本

安装User Scripts插件
下载 pem conf 将pem证书跟conf文件移动到unraid中，如我存放的路径/mnt/user/appdata/ca/
在User Scripts插件中新建名为emby的脚本,并设置为At First Array Start Only

点击EDIT SCRIPTS加入以下内容

!/bin/bash
cp -rf /mnt/user/appdata/ca/default.conf /etc/nginx/conf.d/default.conf
nginx -s reload

APPLY保存 再点击RUN SCRIPTS运行一次脚本

方法2 docker一键伪站 推荐

已制作docker版伪站，下载设置IP后运行即可
https://hub.docker.com/r/s1oz/embyplus

docker pull s1oz/embyplus:latest

额外提一点，伪站基于macvlan，所以emby如果是host模式将无法使用，貌似unriad可设置通信，但貌似会有arp欺诈问题 请将emby也设置为macvlan给与单独IP
我是直接安装在爱快的docker上了,方便的很

方法3 以垃圾群晖为例 建议直接安装第三方已破解的spk更方便 或者docker

群辉套件之类的..已经有垃圾一键脚本了,不过转战docker 已经过期了

wget -N --no-check-certificate "https://raw.githubusercontent.com/s1oz/embyonekey1/master/embyonekey1.sh" && chmod +x embyonekey1.sh && ./embyonekey1.sh

大致内容就是群辉安装Web Station来搭建验证伪站 顺带说一句7.0真垃圾并添加此文件 user.conf
路径为

/usr/local/etc/nginx/conf.d/*-*-*-*

*-*-*-*为建立虚拟主机的随机数字的文件夹
替换后以此命令刷新

nginx -s reload

路由中劫持地址

OP为例 hosts中添加以下信息

  192.168.1.10 mb3admin.com

如果有IPV6的,要么勾选禁止解析 IPv6 DNS 记录

要么就hosts中加入ipv6地址劫持

可能会遇到的问题

核显调用

现在的人,永远就自动照搬,不看官方更新,请不要在go文件中添加

    modprobe i915

因为从beta35开始,官方就给出了此方法

    mkdir /boot/config/modprobe.d
    touch /boot/config/modprobe.d/i915.conf

Unraid 默认会屏蔽显卡驱动,所以通过新建同名文件的方法来解除屏蔽,这样创建后,开机默认会加载核显
请不要在go文件中添加这个了

chmod -R 777 /dev/dri

6.9.1开始,默认会给容器中调用显卡设置为0777权限了
所以只需要选择添加设备,填入/dev/dri即可

Emby服务器没有小黄标

请自行排查mb3admin.com网址劫持到unriad所搭建的验证伪站
一般都是默认解析ipv6域名造成的,禁用或者添加解析即可

windows浏览器没有小黄标

浏览器看片,windows需要安装GMCert.cer证书，安装至受信任的根证书颁发机构
如浏览器还是显示未激活,请先用浏览器访问此地址测试是否正常

https://mb3admin.com/admin/service/registration/validateDevice

如正常显示,重启下电脑试试吧😃
或者直接用电脑客户端

IOS设备无法正常使用

建议用各种工具覆写

#!name=Emby Premiere
#!desc=Emby Premiere Unlock

[Script]
EmbyPremiere.js = type=http-response,pattern=^https?:\/\/mb3admin.com\/admin\/service\/registration\/validateDevice,requires-body=1,max-size=0,script-path=https://gitlab.com/iptv-org/embypublic/-/raw/master/Script/EmbyPremiere.js

[MITM]
hostname = %APPEND% mb3admin.com

Android设备无法正常使用

建议都用手机版本开电视模式,TV版本弱鸡
