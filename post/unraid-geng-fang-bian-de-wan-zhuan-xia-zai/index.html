Unraid更方便的玩转下载

2020-11-22 2 min read # docker # nas # unraid

为什么要这样做

9102年了,谁还没蹄子呢👺
下载工具一般也都是直接安装在NAS上,也没做分离
但是既然用了蹄子,肯定就不能进行p2p下载了👎
应该大部分基友都是搞个OP来煮蹄子吧
lean的只有煮与不煮
要么就是直接不给NAS吃蹄子,docker呀之类的,刮削之类呀,就蛮受影响的
Lienol的可以分用啥方法煮
要么都是给NASGFW模式但是我感觉这蛮影响体验的,大部分好使的,但是不在名单内的网址就跟上面一样受到影响了,所以...唉
既然这样,那就只能靠macvlan了
让NAS与下载工具分别拥有单独IP,这样给NAS吃蹄子,不给下载工具吃蹄子
问题就完美解决了

如何使用

a.Unraid为例

Unraid最简单了,我们先来看下Docker所使用的网络
在终端中输入

docker network ls


看~br0就是,所以我们不需要手动去创建
我们可以直接编辑需要更改的容器

在网络类型中直接选择上面显示的带有macvlan标识的br0
然后输入想要设置的同网段的ip
这样这个容器启动后就会被分配为这个ip
容器也不需要如在Bridge模式一样再次映射端口
相当于弱鸡版本的host模式,容器默认端口多少就是多少
直接ip+端口即可欢快的访问了访问

b.群晖为例

群晖跟Unraid相比,就是默认没有macvlan接口
所以需要多几条命令来建立接口
然后再通过命令生成容器设置ip信息
关于群晖使用,看这吧>bsdcpp<大佬已经给出了命令跟实例了,对照着抄即可

收尾

现在容器已经获取到了独立的IP了👍
那为了愉快的使用,我们直接在蹄子的访问控制中给
NAS中国列表以外
下载工具不代理
这样
NAS刮削,下载,访问都畅通无阻了
下载工具随你上传下载,不走蹄子流量,再也不用担心被ban
